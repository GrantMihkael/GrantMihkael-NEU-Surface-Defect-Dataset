# v0.1 Release Commands

Run these from the project root.

## 1) Review changes
```bash
git status
```

## 2) Stage all Week 1 files
```bash
git add .
```

## 3) Create Week 1 commit
```bash
git commit -m "v0.1: proposal setup, week-1 scaffolds, ethics/model card docs"
```

## 4) Push branch
```bash
git push origin main
```

## 5) Create and push tag
```bash
git tag -a v0.1 -m "v0.1 - Proposal and Setup"
git push origin v0.1
```

## 6) Create GitHub Release
Use tag `v0.1` and copy release notes from `docs/release-notes-v0.1.md`.

If GitHub CLI is installed:
```bash
gh release create v0.1 --title "v0.1 - Proposal and Setup" --notes-file docs/release-notes-v0.1.md
```

## 7) Verify release contents
- Check that README, proposal docs, ethics statement, and model card are visible.
- Confirm tag `v0.1` points to the Week 1 commit.
